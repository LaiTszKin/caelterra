### P2 — Requirement Risk

#### #34 D2: `survey` has no default result cap, risk of excessive output on large directories

- **Files**: `packages/tools/codegraph/lib/cmd-survey.ts:47-162`
- **Dimension**: Spec Implementation Omission
- **Requirements**: D2

**Description**: The Error and Edge Cases section of the discovery spec requires both `explore` and `survey` to have a default result limit (suggesting ~50 symbols) to prevent excessive CLI output. `explore` implements a hardcoded limit of 10 symbols (`cmd-explore.ts:20`), but `survey` has no limit at all — it scans every file in the target directory and returns all symbols, their callers, callees, and relationships. On a large directory (100+ files), the output can exceed thousands of lines, making it unsuitable for LLM consumption and overwhelming the terminal.

The `explore` limit of 10 (vs spec's suggestion of ~50) is acceptable as the spec uses approximate language ("如 50 個符號"), and 10 is more conservative for a deep-dive command. However, the complete absence of any cap on `survey` output means the spec-required safeguard is missing entirely.

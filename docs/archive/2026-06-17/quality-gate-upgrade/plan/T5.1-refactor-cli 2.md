# Worker Prompt: T5.1-refactor-cli

- **Source task**: Batch 5 — Refactor packages/cli/ and bin/ for strict compliance

---

## 1. Mission & Rules

### Mission

Fix all ESLint `strict-type-checked` violations and TypeScript strict flag violations in `packages/cli/` and `bin/` source files. These are purely mechanical type-safety fixes — no behavioral changes.

### Rules

- Only modify `.ts` source files in `packages/cli/` and `bin/`
- Do not change any runtime logic, function signatures that affect callers, or test assertions
- Prefer `unknown` over `any` when adding type annotations
- Add `// eslint-disable-next-line <rule> -- <reason>` comments ONLY for genuine false positives (e.g., third-party API with incomplete types)

### Known Violation Patterns (from DESIGN.md §8)

1. **Indexed access without undefined guard**: `obj[key]` or `arr[i]` patterns. Fix with `if (val !== undefined)` guard or `!` assertion only when provably safe.
2. **`any` type usage in function parameters**: Replace with `unknown` + narrowing, or explicit types like `Record<string, unknown>`, `string[]`, etc.
3. **`!!` double-negation**: Remove extra `!` — autofix may have resolved these.
4. **Optional property initialization with `undefined`**: Change `prop?: T` to `prop: T | undefined` when field is always present.
5. **`process.stdout.isTTY` without explicit guard**: Add explicit `=== true` or `!== undefined` check.

---

## 3. Tasks

### Task — Fix all remaining violations

1. Run `npx eslint packages/cli/ bin/ --cache` to see current violations
2. For each violation, fix according to the pattern:
   - **no-unsafe-argument / no-unsafe-assignment / no-unsafe-call / no-unsafe-member-access / no-unsafe-return**: Replace `any` with `unknown` and add proper type guards, or add explicit type annotations
   - **no-unnecessary-condition**: Remove unnecessary truthiness checks on values that are always truthy
   - **no-unnecessary-type-assertion**: Remove redundant `as Type` casts
   - **prefer-optional-chain**: Replace `a && a.b` with `a?.b`
   - **prefer-nullish-coalescing**: Replace `a || b` with `a ?? b` when the intent is nullish, not falsy
   - **strict-boolean-expressions**: Make boolean conditions explicit
   - **no-unused-vars**: Remove or prefix with `_`
   - **TS strict — noUncheckedIndexedAccess**: Add type guards or non-null assertions
   - **TS strict — exactOptionalPropertyTypes**: Adjust optional property definitions
   - **TS strict — noPropertyAccessFromIndexSignature**: Use bracket notation `obj[key]` instead of `obj.key` for index-signature types
3. After fixing all violations, re-run eslint to verify

### Output

When done, report back:

- **Files modified**: List all files changed
- **Change summary**: Number and types of violations fixed
- **Test results**: `pnpm build` and `pnpm lint` for the CLI package
- **Risks or concerns**: Report any violations that could not be fixed without behavioral changes

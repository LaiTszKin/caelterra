# Worker Prompt: T5.2-refactor-utils

- **Source task**: Batch 5 — Refactor packages/tool-utils/, packages/tool-registry/, packages/tui/ for strict compliance

---

## 1. Mission & Rules

### Mission

Fix all ESLint `strict-type-checked` violations and TypeScript strict flag violations in `packages/tool-utils/`, `packages/tool-registry/`, and `packages/tui/` source files. Mechanical type-safety fixes only.

### Rules

- Only modify `.ts` source files in the three allowed directories
- Do not change any runtime logic
- Prefer `unknown` over `any`
- Only use `eslint-disable` comments for genuine false positives

### Known Violation Patterns (from DESIGN.md §8)

1. **Indexed access without undefined guard**: Fix with `if (val !== undefined)` guard or safe access patterns
2. **`any` type usage**: Replace with `unknown` + narrowing or explicit interface types
3. **`catch` variables**: Ensure catch variables are typed (`catch (err: unknown)` — already implicit in TS strict, but explicit annotations may help eslint)
4. **`!!` double-negation**: Remove extra negation
5. **Optional property vs `T | undefined`**: Fix for `exactOptionalPropertyTypes`
6. **Bracket notation for index signatures**: Fix for `noPropertyAccessFromIndexSignature`

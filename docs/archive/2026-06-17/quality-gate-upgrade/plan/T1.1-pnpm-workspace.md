# Worker Prompt: T1.1-pnpm-workspace

- **Source task**: Batch 1 ‚Äî Create pnpm workspace config and update root package.json

---

## 1. Mission & Rules

### Mission

Create `pnpm-workspace.yaml` matching the existing npm workspace patterns, and update root `package.json` to remove the `workspaces` field and add the `packageManager` pin.

### Rules

- Do not change any dependency versions or add/remove packages
- Do not modify any other files beyond the two listed in Allowed Files
- Preserve the exact existing workspace membership (patterns: `packages/*`, `packages/tools/*`)

---

## 2. Context

### Input Files

- `package.json` ‚Äî Root workspace declaration and metadata (currently has `workspaces`, needs `packageManager`)

### Background

The repo currently uses npm workspaces declared via the `workspaces` field in root `package.json`. After pnpm migration, the workspace membership must be declared in a separate `pnpm-workspace.yaml` file. The `workspaces` field must be removed from `package.json` since pnpm ignores it. A `packageManager` field pins the pnpm version for tools like Corepack.

---

## 3. Tasks

### Task A ‚Äî Create `pnpm-workspace.yaml`

Create a new file at the repository root with the following content:

```yaml
packages:
  - 'packages/*'
  - 'packages/tools/*'
```

### Task B ‚Äî Update `package.json`

1. Locate the `"workspaces"` field (lines 16-19 in the current file)
2. Remove the entire `workspaces` array (the field and both members)
3. After the `"type": "module"` line (currently line 15), add:

```json
  "packageManager": "pnpm@11.6.0",
```

4. Keep all other fields exactly as-is

### Output

When done, report back:

- **Files modified**: `pnpm-workspace.yaml` (created), `package.json` (modified)
- **Change summary**: pnpm workspace config created, workspaces field removed, packageManager pinned
- **Test results**: `pnpm install` should complete without errors
- **Risks or concerns**: None

---

## 4. Verification

1. Run: `pnpm install`
   - Expected: completes with no errors, `pnpm-lock.yaml` is generated
2. Run: `cat pnpm-workspace.yaml`
   - Expected: shows the two package patterns
3. Run: `grep -c '"workspaces"' package.json`
   - Expected: exit code 1 (no match ‚Äî workspaces field removed)

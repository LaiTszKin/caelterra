# Worker Prompt: T5.5-refactor-scripts-tests

- **Source task**: Batch 5 — Refactor scripts/ and test/ files for strict compliance

---

## 1. Mission & Rules

### Mission

Fix all ESLint `strict-type-checked` violations and Prettier formatting issues in `scripts/*.mjs` and `test/*.test.js` files. Note: ESLint may not lint `.js` files with the same strict rules as `.ts` files, but Prettier will format them.

### Rules

- Only modify `.mjs` and `.js` files in `scripts/` and `test/`
- Do not change any test logic or test assertions
- Do not change any script behavior

- Script files may use `process.env` access patterns that need guards

## 3. Tasks

### Task A — Fix scripts/\*.mjs

1. Run `npx eslint scripts/ --cache --ext .mjs` to see violations
2. Fix violations following the standard patterns (catch typing, `no-unsafe-*`, etc.)
3. Run `npx prettier --check scripts/` to check formatting
4. Apply prettier fixes if needed

### Task B — Check test/\*.test.js formatting

1. Run `npx prettier --check "test/**/*.test.js"` to check formatting
2. Apply prettier fixes if needed
3. Note: ESLint strict rules are relaxed for test files (configured in eslint.config.mjs), so only fix what eslint flags

### Output

When done, report back:

- **Files modified**: List all files changed
- **Change summary**: Types and counts of violations fixed
- **Test results**: `pnpm test` should pass
- **Risks or concerns**: Report any test that fails after changes

---

## 4. Verification

1. Run: `npx eslint scripts/ --cache --ext .mjs`
   - Expected: 0 errors
2. Run: `npx prettier --check "scripts/**/*.mjs" "test/**/*.test.js"`
   - Expected: All matched files use Prettier code style
3. Run: `pnpm test 2>&1 | tail -10`
   - Expected: All tests passing (or no regressions from pre-refactoring state)

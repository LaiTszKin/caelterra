# Worker Prompt: T5.3-refactor-codegraph-eval

- **Source task**: Batch 5 — Refactor packages/tools/codegraph/ and packages/tools/eval/ for strict compliance

---

## 1. Mission & Rules

### Mission

Fix all ESLint `strict-type-checked` violations and TypeScript strict flag violations in the two most complex tool packages: `codegraph` and `eval`. These packages are the most challenging due to their size, complexity, and use of third-party APIs (codegraph npm package, Judge Model API).

### Rules

- Only modify `.ts` source files in the two allowed directories
- Do not change any runtime logic
- Prefer `unknown` over `any`
- For third-party APIs with incomplete types (`@colbymchenry/codegraph`, Judge Model), local type augmentations are preferred over `eslint-disable`

### Known Violation Patterns

- **process.stdout.isTTY**: Guard with explicit `=== true` check
- **`!!options.index`**: The `!!` will be autofixed by eslint --fix
- **Third-party API types**: `@colbymchenry/codegraph` may return `any` types — create local type wrappers rather than spreading `any`
- **Array<T> generic**: May be normalized by prettier/eslint
- **Indexed access**: Bracket access on arrays or records needs undefined guards
- **`any` from external API responses**: Judge model responses need typed wrappers or `unknown` casts

# Worker Prompt: T1.2-ci-pnpm

- **Source task**: Batch 1 — Update all CI workflows for pnpm compatibility

---

## 1. Mission & Rules

### Mission

Update all four GitHub Actions workflow files to use pnpm instead of npm: replace `npm ci` with `pnpm install --frozen-lockfile`, add the `pnpm/action-setup` step, and update the cache key from `npm` to `pnpm`.

### Rules

- Do not change any other CI behavior (node versions, OS matrix, test commands, publish config, triggers)
- All four workflow files must be updated consistently
- Keep `shell: bash` settings and the `COVERAGE: 'true'` env vars unchanged

---

## 2. Context

### Input Files

- `.github/workflows/test.yml` — Test workflow (push/PR, matrix: ubuntu+windows, node 22)
- `.github/workflows/publish-npm.yml` — Publish workflow (release event, ubuntu, node 24)
- `.github/workflows/eval.yml` — Eval workflow
- `.github/workflows/skill-validation.yml` — Skill validation workflow

### Background

pnpm requires a setup action (`pnpm/action-setup@v6`) before the `actions/setup-node` step to install pnpm itself. The `cache` key in `setup-node` must be `'pnpm'` instead of `'npm'`. The install command changes from `npm ci` to `pnpm install --frozen-lockfile`. Build/test commands change from `npm run build` to `pnpm run build` (or just `pnpm build`).

---

## 3. Tasks

### For each of the four `.github/workflows/*.yml` files:

1. After the `actions/checkout@v4` (or @v5) step, add:

```yaml
- uses: pnpm/action-setup@v6
  with:
    version: 11
```

2. In the `actions/setup-node` step, change:
   - `cache: 'npm'` → `cache: 'pnpm'`

3. Replace all `npm ci` commands with `pnpm install --frozen-lockfile`

4. Replace all `npm run build` with `pnpm run build` (or `pnpm build`)

5. Replace all `npm test` with `pnpm test`

6. For `publish-npm.yml`: replace `npm publish --access public` with `pnpm publish --access public`

### Output

When done, report back:

- **Files modified**: List the 4 workflow files changed
- **Change summary**: pnpm action-setup added, cache key changed, npm→pnpm commands migrated
- **Test results**: N/A (CI-only change, verified by coordinator)
- **Risks or concerns**: Ensure `pnpm publish` is used (not `npm publish`) — critical for `workspace:*` protocol resolution

---

## 1. Mission & Rules

### Mission

### Rules

- Do not modify `cli.js`, `index.ts`, or any test files

### Background

`buildArchitectureHelpPage(verb, subverb)` in `cli-help.js` generates help for:
1. **Top-level help** (lines 729-782): When called with no arguments. Lists all verbs in notes + usage examples.
2. **Family help** (line 789): For `feature`, `submodule`, etc. `familyPages` dict.
3. **Action help** (line 785): For `feature:add`, `submodule:remove`, etc. `actionPages` dict.
4. **Verb help** (lines 793-998): Individual switch cases for `open`, `diff`, `render`, `validate`, `scan`, `undo`, `merge`.

The change must:
- Keep familyPages and actionPages intact (backward compat for direct calls)
- Keep verb switch cases for `open`, `diff`, `render`, `validate`, `scan`, `undo`, `merge` intact (they still need help when called directly)
- Add new help entries for `add` and `remove` verbs
- Update the top-level summary to mention only the 6 public verbs

## 3. Tasks

### `packages/tools/architecture/index.test.ts` — Update TS handler tests

1. Open `packages/tools/architecture/index.test.ts`
2. Find tests related to `apply` (REGTEST-15, REGTEST-16, REGTEST-17)

6. Add a new test case verifying that `architectureHandler(['apply', '...'], mockContext)` no longer gets intercepted (returns non-zero from cli.js)

### `test/atlas-cli.test.js` — Add integration tests for add/remove

1. Open `test/atlas-cli.test.js`
2. Add a test suite at the end (or in an appropriate location) covering:
   - `apltk architecture add feature test-feat --depends-on other` creates a feature entry
   - `apltk architecture add module test-mod --part-of test-feat` creates a submodule
   - `apltk architecture add relation test-mod --data-flow-to other-mod` creates an edge
   - `apltk architecture add` batch mode with multiple entities
   - `apltk architecture remove feature test-feat` removes a feature
   - Error cases: unknown entity type, missing required flags like `--part-of` for module
   - Spec mode: `add --spec <dir>` writes to overlay, not baseline

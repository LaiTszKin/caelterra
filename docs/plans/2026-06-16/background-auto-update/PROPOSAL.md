# Proposal: 背景自動更新

- **Date**: 2026-06-16
- **Source**: 由 `discuss` 技能通過結構化對話釐清需求後產出

---

## 1. 功能範圍

### 要做的事

- 在用戶安裝 Apollo Toolkit 之後，預設啟用背景自動更新。
- 背景自動更新只覆蓋 Apollo Toolkit 管理和安裝的技能/工作流。
- 背景自動更新透過定時背景任務執行，預設每天檢查一次。
- 支援 macOS、Linux、Windows 三個作業系統。
- 自動檢查並套用非破壞性更新，讓用戶不需要手動維護技能/工作流版本。
- 若本地技能/工作流被用戶手動修改，背景更新仍直接覆蓋成本地最新版本。
- 提供 CLI 命令讓用戶關閉、重新開啟、查詢目前背景自動更新狀態。
- 即使用戶關閉背景自動更新，仍允許用戶手動更新 Apollo Toolkit 管理的技能/工作流。
- 更新失敗時不打斷用戶目前工作，但需要留下簡短提醒或日誌方便排查。

### 明確不做的事

- 不更新 Apollo Toolkit CLI 本身。
- 不保護或合併用戶手動修改過的技能/工作流內容。
- 不要求每次啟動 CLI 或每次啟動工作流時都同步檢查更新。
- 不把關閉自動更新解讀為禁止手動更新。
- 不以完全靜默的方式處理更新失敗。

---

## 2. 使用者場景

### 目標使用者

Apollo Toolkit 的一般用戶：他們已經安裝 Apollo Toolkit，並希望已安裝的技能/工作流可以持續保持最新，而不需要定期手動執行更新命令。

### 典型流程

**場景 A：安裝後自動保持最新**

1. 用戶安裝 Apollo Toolkit。
2. 系統預設啟用背景自動更新，並建立定時背景任務。
3. 背景任務每天檢查一次 Apollo Toolkit 管理的技能/工作流是否有可用更新。
4. 若有非破壞性更新，系統自動套用到本地安裝位置。
5. 用戶之後使用工作流時，通常已經取得較新的技能/工作流版本。

**場景 B：用戶關閉背景自動更新**

1. 用戶不希望系統在背景自動更新。
2. 用戶透過 CLI 命令關閉背景自動更新。
3. 系統保存關閉狀態，並停止定時背景任務。
4. 用戶仍然可以透過手動命令更新 Apollo Toolkit 管理的技能/工作流。

**場景 C：用戶重新開啟背景自動更新**

1. 用戶希望恢復自動更新。
2. 用戶透過 CLI 命令重新開啟背景自動更新。
3. 系統保存開啟狀態，並恢復定時背景任務。
4. 後續更新再次按預設每日檢查節奏執行。

**場景 D：本地內容被覆蓋**

1. 用戶曾手動修改 Apollo Toolkit 管理的技能/工作流。
2. 背景更新檢查到該技能/工作流有新版本。
3. 系統直接用最新版本覆蓋本地版本。
4. 被手動修改過的本地內容不會被保留、合併或提示確認。

### 成功標準

- 用戶安裝後，即使不手動更新，也能在合理時間內取得最新的 Apollo Toolkit 技能/工作流。
- 用戶可以清楚控制背景自動更新是開啟還是關閉。
- 用戶關閉背景自動更新後，仍然可以手動更新。
- 更新失敗不會阻塞或中斷用戶正在進行的工作。

### 失敗處理

- 若背景更新失敗，不中斷用戶目前工作。
- 若背景更新失敗，需要留下簡短提醒或日誌，讓用戶之後能理解更新沒有完成。
- 若用戶查詢自動更新狀態，CLI 應能清楚顯示目前是開啟或關閉。

---

## 3. 約束條件

- **預設行為**：背景自動更新預設開啟。
- **更新頻率**：預設每天檢查一次。
- **平台範圍**：需要支援 macOS、Linux、Windows。
- **可控性**：用戶必須可以透過 CLI 關閉、重新開啟、查詢背景自動更新狀態。
- **狀態保存**：開啟或關閉狀態需要被保存，不只是單次命令有效。
- **覆蓋策略**：若本地技能/工作流被手動修改，背景更新仍直接覆蓋成本地最新版本；這是已確認的產品行為。
- **信任與透明度**：由於功能預設開啟，且會建立背景任務並覆蓋本地內容，用戶必須有明確的關閉和查詢能力。

---

## 4. 業務價值

### 解決的問題

Apollo Toolkit 的技能/工作流會持續演進；如果用戶需要手動維護版本，容易使用過期工作流，增加維護成本和使用風險。

### 為什麼現有方案不夠

- 手動更新依賴用戶記得執行命令，容易被忽略。
- 用戶可能不知道技能/工作流已有新版本。
- 若工作流更新包含修正或改善，未自動更新會讓用戶繼續使用舊行為。

### 成功指標

- 用戶安裝後，不需要手動更新，也能在合理時間內取得最新技能/工作流。
- CLI 能清楚顯示背景自動更新狀態，並能開啟或關閉該功能。
- 背景更新失敗不影響用戶的主要 CLI 使用流程。

---

## 5. 需求摘要

- 安裝後預設啟用背景自動更新，讓 Apollo Toolkit 安裝的技能/工作流保持最新。
- 自動更新透過定時背景任務執行，預設每天檢查一次，支援 macOS、Linux、Windows。
- 背景更新只覆蓋 Apollo Toolkit 管理的技能/工作流，不負責更新 CLI 本身。
- 背景更新會自動套用非破壞性更新；若本地技能被手動修改，仍直接覆蓋成本地最新版本。
- 更新失敗不應打斷使用者目前工作，但需要留下簡短提醒或日誌方便排查。
- CLI 需要提供關閉、重新開啟、查詢目前自動更新狀態的能力。
- 即使用戶關閉背景自動更新，仍應允許手動更新。

---

## 6. 剩餘未解決的問題

- 「非破壞性更新」的具體判定規則尚未定義，需在後續 spec 或 design 階段細化。
- 各作業系統上的背景任務建立、移除和恢復方式尚未定義，需在 design 階段細化。
- 更新失敗時的提醒形式和日誌位置尚未定義，需在 design 階段細化。

# Regression Test Worker Prompt: REGTEST-02-windows-schtasks-quoting

- **Related fix**: FIX-02 — Windows schtasks quoting

---

## 1. Mission & Rules

### Mission

Add a regression test proving Windows `schtasks /TR` preserves runner command arguments that contain spaces.

### Context

FIX-02 changes the Windows command string generated in `registerAutoUpdateTask()`. The unfixed code uses `options.runnerCommand.join(' ')`, so a path like `C:\Program Files\nodejs\node.exe` or `C:\Users\Jane Doe\.apollo-toolkit` is split incorrectly in the scheduled task action.

### Rules

- Only create or modify test files — never modify source code.
- The test must fail on the unfixed code and pass after FIX-02.
- Follow existing `node:test` style in `test/cli/auto-update-scheduler.test.js`.
- Do not mutate real OS schedulers; use the existing fake executor pattern.
- Workers are leaf nodes — do not spawn sub-workers.

## 2. Context

### Input Files

- Fix-related file: `packages/cli/auto-update-scheduler.ts`.
- Existing test file: `test/cli/auto-update-scheduler.test.js`.
- Worker prompt: `docs/plans/2026-06-16/background-auto-update/fix/FIX-02-windows-schtasks-quoting.md`.

### Test Design

- **Test ID**: REGTEST-02
- **Type**: Unit test.
- **Location**: `test/cli/auto-update-scheduler.test.js`.
- **Scenario**: GIVEN Windows platform and a runner command containing spaces in node path, CLI path, and toolkit home WHEN `registerAutoUpdateTask()` builds the `schtasks /Create` call THEN the `/TR` argument contains quoted path arguments so each spaced path remains one command argument.
- **Oracle**: On unfixed code, `/TR` is an unquoted plain join and does not contain quoted spaced paths. After FIX-02, `/TR` contains quoted spaced path arguments.

8. Also assert `tr` still includes `auto-update run --home`.
9. Do not require an exact full string if escaping details make the test brittle; assert the meaningful quoted substrings.

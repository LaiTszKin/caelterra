# Regression Test Worker Prompt: REGTEST-01-scheduled-runner-entrypoint

- **Related fix**: FIX-01 — scheduled runner entrypoint

---

## 1. Mission & Rules

### Mission

Add a regression test proving the auto-update scheduler command registered by the CLI targets the executable bin wrapper, not the non-executing CLI library module.

### Context

FIX-01 changes the command target passed to `buildRunnerCommand()`. The unfixed code schedules `packages/cli/dist/index.js`, and `node packages/cli/dist/index.js auto-update run ...` exits without dispatching. The test must fail on that unfixed path and pass only when the command includes `dist/bin/apollo-toolkit.js`.

### Rules

- Only create or modify test files — never modify source code.
- The test must fail on the unfixed code and pass after FIX-01.
- Follow existing `node:test` style in `test/cli/*.test.js`.
- Do not mutate real OS schedulers.
- Workers are leaf nodes — do not spawn sub-workers.

---

## 2. Context

### Input Files

- Fix-related files: `packages/cli/index.ts`, `dist/bin/apollo-toolkit.js`, `bin/apollo-toolkit.ts`.
- Existing test files: `test/cli/interactive-paths.test.js`, `test/cli/dispatch-table.test.js`, `packages/tools/codegraph/lib/cmd-init.test.ts`.
- Worker prompt: `docs/plans/2026-06-16/background-auto-update/fix/FIX-01-scheduled-runner-entrypoint.md`.

### Test Design

- **Test ID**: REGTEST-01
- **Type**: Integration/unit hybrid with module mocking.
- **Location**: Prefer a new `test/cli/auto-update-cli-wiring.test.js`; alternatively extend an existing CLI test file if that matches local style.
- **Scenario**: GIVEN the CLI `auto-update enable` command and a mocked scheduler adapter WHEN `run(['auto-update', 'enable', '--home', tempHome], context)` executes THEN the runner command passed to `registerAutoUpdateTask()` contains `dist/bin/apollo-toolkit.js` and does not contain `packages/cli/dist/index.js`.
- **Oracle**: On unfixed code, captured `runnerCommand[1]` is the CLI library module path and the assertion fails. After FIX-01, captured `runnerCommand[1]` ends with `dist/bin/apollo-toolkit.js` and the assertion passes.

## 3. Tasks

1. Create or update `test/cli/auto-update-cli-wiring.test.js`.
2. Use `node:test` and `node:assert/strict`.
3. Use the existing `mock.module()` pattern from `packages/tools/codegraph/lib/cmd-init.test.ts` before importing `../../packages/cli/dist/index.js`:
   - Mock `../../packages/cli/dist/auto-update-scheduler.js`.
   - Export real `buildRunnerCommand` if convenient, or provide an equivalent fake that returns the command array.
   - Mock `registerAutoUpdateTask()` to capture its `options.runnerCommand` and return `{ registered: true, platform: 'darwin', message: 'mocked' }`.
   - Mock `unregisterAutoUpdateTask()` and `getAutoUpdateTaskStatus()` only if needed for module import completeness.
4. Import `run` from `../../packages/cli/dist/index.js` after the mock is registered.
5. Provide a `context` with:
   - `sourceRoot` set to the repository root path so the expected bin path resolves to `<repo>/dist/bin/apollo-toolkit.js`.
   - fake stdout/stderr objects that collect writes.
   - minimal env with `HOME` set to a temp directory.
6. Execute `run(['auto-update', 'enable', '--home', tempHome], context)`.
7. Assert:
   - return code is `0`.
   - the mocked scheduler was called once.
   - captured `runnerCommand[1]` equals or ends with `dist/bin/apollo-toolkit.js`.
   - captured `runnerCommand[1]` does not end with `packages/cli/dist/index.js`.
8. Keep the test deterministic and avoid invoking `launchctl`, `systemctl`, or `schtasks`.

### Output

- **Oracle confirmed**: fails before FIX-01 and passes after
- **Risks or concerns**: or `None`

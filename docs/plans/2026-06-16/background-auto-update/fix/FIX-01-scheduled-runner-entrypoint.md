# Fix Worker Prompt: FIX-01-scheduled-runner-entrypoint

- **Related issue**: P1-1 — Scheduled task invokes the CLI library module, which does not execute `run()` when called as a script

---

## 1. Mission & Rules

### Mission

Make every scheduled auto-update task invoke the real Apollo Toolkit executable entrypoint so OS scheduler runs actually dispatch `auto-update run`.

### Context

REPORT.md flags this as a Spec implementation deviation for Req 1 and Req 2. The current install and enable flows build the runner command with `fileURLToPath(import.meta.url)` inside `packages/cli/index.ts`, which points to the CLI library module. The real executable wrapper is `dist/bin/apollo-toolkit.js` and calls `run(process.argv.slice(2), { sourceRoot })`.

### Rules

- Follow the Scope in Section 5 — only modify files listed as Allowed.
- Preserve existing test semantics — do not weaken, skip, or remove existing tests.
- Do not change scheduler platform behavior except the command target needed for this issue.
- Do not add dependencies.
- Workers are leaf nodes — do not spawn sub-workers.

---

## 2. Context

### Input Files

- `packages/cli/index.ts` — read the `run()` function, auto-update enable/status/disable flow, and install auto-enable flow.
- `bin/apollo-toolkit.ts` — read the executable wrapper shape and sourceRoot calculation.
- `dist/bin/apollo-toolkit.js` — confirm the built executable path used by the published package.
- `packages/cli/auto-update-scheduler.ts` — read `buildRunnerCommand()` to understand command inputs.
- `docs/plans/2026-06-16/background-auto-update/REPORT.md` — read P1-1.

### Root Cause

`packages/cli/index.ts` is an importable library module. It exports `run()` but has no top-level script invocation. The scheduler command currently points at that module, so `node packages/cli/dist/index.js auto-update run --home ...` exits without invoking the CLI. The command target must be the bin wrapper or an equivalent executable entrypoint.

## 3. Tasks

### `packages/cli/index.ts` — use executable bin path for scheduled runner commands

1. Open `packages/cli/index.ts`.
2. Locate every auto-update runner command construction:
   - `action === 'enable'` around L347-L356.
   - `action === 'disable'` around L365-L373.
   - `action === 'status'` around L387-L395.
   - install auto-enable around L518-L527.
3. Add a local helper near `readPackageJson()` or near `run()` that derives the bin wrapper path from the already computed `sourceRoot`, for example a function that returns `path.join(sourceRoot, 'dist', 'bin', 'apollo-toolkit.js')`.
4. Replace the current `const cliPath = fileURLToPath(import.meta.url);` runner-command target in the four locations with the executable bin wrapper path.
5. Keep `nodePath = process.execPath`, `toolkitHome`, action names, and `buildRunnerCommand()` unchanged.
6. Leave the top-level `run()` export contract unchanged.

# Fix Worker Prompt: FIX-02-windows-schtasks-quoting

- **Related issue**: P1-2 — Windows scheduled task command is joined without quoting argument paths

---

## 1. Mission & Rules

### Mission

Make the Windows `schtasks /TR` action preserve runner command arguments that contain spaces, especially Node path, CLI path, and toolkit home path.

### Context

REPORT.md flags this as a Spec implementation deviation for Req 2. DESIGN.md marks quoted runner command handling as non-negotiable for Windows scheduled tasks. The current implementation uses `options.runnerCommand.join(' ')` in the Windows branch of `registerAutoUpdateTask()`.

### Rules

- Follow the Scope in Section 5 — only modify files listed as Allowed.
- Preserve macOS and Linux scheduler behavior.
- Preserve existing tests — do not weaken, skip, or remove them.
- Do not introduce shell execution or new dependencies.
- Workers are leaf nodes — do not spawn sub-workers.

---

## 2. Context

### Input Files

- `packages/cli/auto-update-scheduler.ts` — read `buildRunnerCommand()` and the Windows branch of `registerAutoUpdateTask()`.
- `test/cli/auto-update-scheduler.test.js` — read current scheduler command-shape tests for style and expectations.
- `docs/plans/2026-06-16/background-auto-update/references/os-scheduled-tasks.md` — read Windows `schtasks` command-shape notes.
- `docs/plans/2026-06-16/background-auto-update/REPORT.md` — read P1-2.

### Root Cause

The Windows scheduler action is a single `/TR` string interpreted by `schtasks`. Plainly joining argument array entries with spaces destroys argument boundaries when any path contains spaces. The command string must quote arguments that need quoting and escape embedded double quotes.

---

## 3. Tasks

### `packages/cli/auto-update-scheduler.ts` — quote Windows `/TR` command arguments

1. Open `packages/cli/auto-update-scheduler.ts`.
2. Locate `registerAutoUpdateTask()` and the `case 'win32'` branch around L274-L283.
3. Add a small helper in the scheduler module, near the existing task definition helpers:
   - Input: one command argument string.
   - Output: unchanged string when no quoting is needed.
   - Output: double-quoted string when the argument contains whitespace or double quotes.
   - Escape embedded double quotes so the resulting `/TR` string remains one valid action command.
4. Replace `const commandStr = options.runnerCommand.join(' ');` with a join over the Windows quoting helper.
5. Keep `/Create`, `/SC DAILY`, `/TN ApolloToolkitAutoUpdate`, `/ST 09:00`, and `/F` unchanged.
6. Do not change Linux `ExecStart` or macOS launchd plist generation in this worker.

### Output

When done, report back to the coordinator:
- **Files modified**: `packages/cli/auto-update-scheduler.ts`
- **Change summary**: Windows `/TR` now preserves argument boundaries for paths with spaces
- **Test results**: commands run and pass/fail
- **Risks or concerns**: any unresolved Windows quoting assumptions

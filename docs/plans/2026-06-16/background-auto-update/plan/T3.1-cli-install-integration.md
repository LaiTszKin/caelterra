# Worker Prompt: T3.1-CLI install integration

- **Source task**: T3.1 CLI entry point and install flow integration

---

## 1. Mission & Rules

### Mission

Wire the new auto-update command surface into the CLI entry point and ensure install flow enables background auto-update by default without re-enabling an explicitly disabled state.

### Rules

- Follow the Scope in Section 5 — only modify files listed as Allowed.
- Do not change parser definitions beyond import/use wiring unless a compile error forces a minimal correction.
- Do not change scheduler or runner internals; those are separate tasks.
- Workers are leaf nodes — do not spawn sub-workers.

### Input Files

- `test/cli/dispatch-table.test.js`, `test/cli/interactive-paths.test.js`, `test/installer.test.js` — integration test style.

### Background

The install flow already updates the CLI package interactively. This task must keep that behavior intact while also enabling managed-skill background auto-update by default. If the user has explicitly disabled auto-update, install should preserve that state.

---

## 3. Tasks

### `packages/cli/index.ts` — add auto-update dispatch

1. Open `packages/cli/index.ts`.
2. Import the new auto-update parser and controller/run helpers created by earlier workers.
3. Extend `parseArguments` dispatch so `auto-update` is recognized as a top-level command.
4. Update help handling so `apltk auto-update --help` routes to the new help text.
5. In the install flow, after successful skill installation, ensure auto-update is enabled by default unless config already says disabled.
6. If default enablement fails, report it visibly and preserve installed skills.
7. Do not alter the existing package self-update check semantics.

### `packages/cli/installer.ts` — minimal install hook if needed

1. Open `packages/cli/installer.ts` only if a minimal helper needs to be exported for the install flow.
2. Add only the smallest possible helper needed to query installed toolkit home or managed modes from install context.
3. Do not change skill copy/symlink semantics.

### Tests

1. Extend `test/cli/dispatch-table.test.js` to cover `auto-update` routing.
2. Extend `test/installer.test.js` or `test/cli/interactive-paths.test.js` to assert:
   - Install enables auto-update by default when no disabled config exists.
   - Existing disabled config is preserved.
3. Keep assertions focused on observable CLI behavior and state helpers, not internal implementation details.

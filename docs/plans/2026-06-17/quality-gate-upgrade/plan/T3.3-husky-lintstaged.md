# Worker Prompt: T3.3-husky-lintstaged

- **Source task**: Batch 3 — Configure husky pre-commit hook with lint-staged

---

## 1. Mission & Rules

### Mission

Update the `.husky/pre-commit` hook file to run `pnpm lint-staged` before each commit.

### Rules

- Do not touch `.husky/` files outside of the pre-commit hook
- husky v9 does NOT need a shell shebang in the hook file
- The hook must be a plain shell command, not Node.js

---

## 2. Context

### Input Files

- `.husky/pre-commit` — Created by `pnpm exec husky init` (contains a default command like `pnpm test`)

### Background

husky v9 uses a `.husky/` directory with hook files. The init command creates an initial `pre-commit` file. We need to replace its content to run lint-staged instead. The hook should be a simple shell command.

Note: `lint-staged` was already configured in `.lintstagedrc.json` (created by the T3.2 worker), so the hook only needs to call `pnpm lint-staged`.

---

## 3. Tasks

### Task — Update `.husky/pre-commit`

1. Open `.husky/pre-commit`
2. Replace the entire file content with:

```
pnpm lint-staged
```

3. Save

That's all that's needed — husky executes the file directly, and the `pnpm` command is available because husky preserves the PATH from the git environment.

### Output

When done, report back:

- **Files modified**: `.husky/pre-commit`
- **Change summary**: Hook now runs `pnpm lint-staged` before each commit
- **Test results**: N/A (verified by manually testing commit behavior)
- **Risks or concerns**: None

---

## 4. Verification

1. Run: `cat .husky/pre-commit`
   - Expected: Contains exactly `pnpm lint-staged` (or `pnpm lint-staged\n`)
2. Run: `chmod +x .husky/pre-commit && .husky/pre-commit`
   - Expected: lint-staged runs with no staged files (should output something like "No staged files found")

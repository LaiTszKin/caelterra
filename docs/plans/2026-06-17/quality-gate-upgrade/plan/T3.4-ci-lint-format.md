# Worker Prompt: T3.4-ci-lint-format

- **Source task**: Batch 3 — Add lint and format-check steps to CI workflows

---

## 1. Mission & Rules

### Mission

Add `pnpm lint` and `pnpm format:check` steps to all CI workflow files, positioned before the test step.

### Rules

- Do not remove or modify any existing CI steps
- Only add the new lint/format steps — do not change the existing pnpm migration changes (already done in T1.2)
- lint step runs BEFORE format-check (faster failure)
- Keep `shell: bash` setting if the surrounding steps use it
- Do not add these steps to `publish-npm.yml` (keep that workflow minimal since tests already run)

---

## 2. Context

### Input Files

- `.github/workflows/test.yml` — Main CI workflow
- `.github/workflows/eval.yml` — Eval workflow
- `.github/workflows/skill-validation.yml` — Skill validation workflow

### Background

Each workflow was already updated in Batch 1 to use pnpm commands. Now we need to add quality gate steps before the test step. This ensures CI fails fast on code quality issues rather than waiting for the full test suite.

The lint step should use `--cache` flag to speed up subsequent runs (the cache is generated on the first CI run and used in later runs on the same branch).

---

## 3. Tasks

### For each workflow file (test.yml, eval.yml, skill-validation.yml):

Insert the following two steps BEFORE the test/run step (typically named "Run tests with coverage" or similar):

```yaml
- name: Lint check
  run: pnpm lint
- name: Format check
  run: pnpm format:check
```

For `test.yml`, insert between the build step and the "Run tests with coverage" step.

For `eval.yml` and `skill-validation.yml`, insert between the build step and the test/run step.

Do NOT modify `publish-npm.yml` — the publish workflow already runs tests before publishing, and adding lint/format there adds unnecessary delay to the publish pipeline.

### Output

When done, report back:

- **Files modified**: List the workflow files updated
- **Change summary**: Added `Lint check` and `Format check` steps before test step in N workflow files
- **Test results**: N/A
- **Risks or concerns**: Ensure the lint step is before format-check for faster failure

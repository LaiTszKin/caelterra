## 技能目標

將 review 技能發現的問題清單（REPORT.md）轉化為可執行的修復方法論（FIX.md）。
分離「規劃如何修復」與「執行修復」兩個階段。

## 驗收條件

- 產出 FIX.md，放置在 spec 目錄下
- FIX.md 包含完整的修復依賴分析、檔案重疊檢測、批次排程、subagent 路由分配
- FIX.md 可直接被 fix 技能消費，無需 fix 再做規劃決策

## 工作流程

### 1. 閱讀輸入文件

完整閱讀以下文件：

- **spec 文檔組**：spec.md、tasks.md、design.md、checklist.md、contract.md
- **REPORT.md**：review 技能產出的問題清單（判決 + P0-P3 問題列表 + 審查維度摘要）

理解 spec 的原始設計意圖與 REPORT.md 中每個問題的本質。

### 2. 閱讀相關代碼

根據 REPORT.md 中標記的檔案路徑，閱讀受影響的代碼。
理解每個問題的實際程式碼上下文，以便制定精確的修復方案。
如果外部環境允許使用 subagents，通過並行調度 subagents 完成代碼閱讀。

### 3. 分析每個問題並制定修復方案

對 REPORT.md 中的每個問題（按 P0 → P1 → P2 → P3 順序）：

- **根因分析**：透過程式碼閱讀確定問題的根本原因
- **修復方案**：描述具體的修改方式（修改哪些檔案、哪些函式、如何修改）
- **驗證方式**：定義修復完成後的驗證命令或手動檢查步驟
- **風險標記**：auth / schema / migration / 外部 API / 無
- **複雜度分類**：
  - **簡單修復**：單一檔案內的明確修改 → 直接編輯
  - **複雜修復**：跨多檔案、需深入理解執行路徑 → 需使用 `systematic-debug`

### 4. 分析修復依賴關係

分析各問題之間的依賴關係：

- **檔案重疊依賴**：多個問題觸及相同檔案的相同區域 → 必須循序
- **邏輯依賴**：修復 B 依賴修復 A 先完成（例如 A 重構了介面，B 才能適配新介面）
- **獨立問題**：無檔案重疊且無邏輯相依 → 可並行處理

### 5. 檢測檔案重疊

對所有修復工作進行檔案重疊檢測：

1. 收集每個問題涉及的檔案清單
2. 比對檔案清單，標記重疊
3. 檔案重疊的問題不得並行修復

### 6. 建立修復批次排程

根據依賴分析和檔案重疊檢測結果，建立修復批次排程：

**批次劃分原則：**
- 優先修復 P0 問題（可能阻斷其他修復的驗證）
- 同一批次內的問題之間：無檔案重疊、無邏輯依賴
- 不同批次之間：前一批次完成並驗證後，才開始下一批次
- 每個批次的產出是可獨立驗證的中間狀態

**排程輸出格式：**
```
Batch 1: [P0-1] ∥ [P0-3]  （獨立 P0，並行修復）
Batch 2: [P0-2] → [P1-1]   （P0-2 是 P1-1 的前置）
Batch 3: [P1-2] ∥ [P2-1] ∥ [P3-1]  （低優先級，並行收尾）
```

### 7. 決定 Subagent 路由（可選）

若存在可並行修復的問題，決定是否需要 subagent 路由：

**觸發條件：**
- 同一批次中有 ≥2 個獨立問題
- 問題涉及 ≥3 個不同檔案
- 修復工作量足夠大（預計修改 >50 行）

**每個 subagent 的定義包含：**
- 負責的問題 ID
- 目標檔案清單
- 禁止修改的檔案（屬於其他 subagent）
- 修復方案摘要
- 驗證命令

若不需要 subagent（問題少、範圍小），標記為「循序修復」即可。

### 8. 定義回歸測試策略

為整個修復計畫定義回歸測試策略：
- 哪些現有測試必須繼續通過
- 是否需要新增回歸測試
- 哪些修復需要 property-based 測試

### 9. 產出 FIX.md

使用 `assets/templates/FIX.md` 模板，填入完整修復計畫。
將 FIX.md 放置在 spec 目錄下（與 REPORT.md 同層）。

## 範例

- "REPORT.md 有 3 個 P0、2 個 P1、1 個 P2" → 分析依賴 → P0 問題獨立無重疊 → Batch 1 並行修復 P0 → Batch 2 修復 P1+P2 → 產出 FIX.md
- "兩個 P0 問題都修改 src/auth.ts" → 檔案重疊 → 必須循序修復 → 排程: Batch 1 修復 P0-1 → Batch 2 修復 P0-2
- "6 個獨立問題分佈在 5 個不同檔案" → 觸發 subagent 路由 → 分配 3 個 subagents → 產出 FIX.md 含完整路由表

## 技能目標

遵照 code review report 的建議方案修正代碼問題。
逐一處理所有發現的問題，修復後必須經過驗證方可提交。

## 驗收條件

- 所有 code review report 中明確列出的 P0-P3 問題都已被完全修復。
- 每個修復都經過驗證（測試通過 / 編譯成功）。
- 無引入新的退化（regression）。

## 工作流程

### 1. 完整閱讀 code review report

完整閱讀 code review report 及相關受影響代碼，理解建議修復方案。
如果外部環境允許使用 subagents，建議通過並行調度 subagents 完成對代碼的深度閱讀。

### 2. 分類修復難度

根據 code review report 中問題的性質，分類修復策略：

- **簡單修復**：單一檔案內的明確修改（如更換 API 呼叫、修正變數名稱、調整條件邏輯）。直接編輯，不需要啟動 `systematic-debug`。
- **複雜修復**：跨多個檔案、需要深入理解執行路徑、或根因不明確的問題。必須使用 `systematic-debug` 技能進行系統性除錯。

### 3. 修復發現的問題

如果外部環境不允許使用 subagents，按照以下流程完成修復工作：
1. 將所有問題按 code review report 的嚴重度排序。
2. 從最高嚴重度的問題開始修復。簡單修復直接編輯代碼；複雜修復使用 `systematic-debug` 技能，直至完成所有建議修復。

若外部環境允許使用 subagents，必須按照以下流程加速修復進度：
1. 讀取審查結果所發現的問題，識別每個問題對應的改進建議。
2. 閱讀審查報告標記的相關檔案。
3. 將審查報告標記的檔案路徑進行**重疊檢查**：
   - 若多個問題觸及**不同檔案**，可安全地平行修復
   - 若多個問題觸及**相同檔案**，必須循序修復以避免衝突
4. 將問題區分為：
   - 獨立問題：與其他問題之間沒有檔案重疊且沒有邏輯相依性
   - 非獨立問題：與其他問題之間存在檔案重疊或邏輯相依性
5. 召喚多個有自己專屬 worktree 的 subagents 對獨立問題進行並行修復。
6. 合併所有獨立問題的修復到當前分支；將非獨立問題按照其相依性切分為多個修復批次，並確保每一個批次之中的任務之間沒有相依性。
7. 按照非獨立問題的修復批次順序，召喚多個有自己專屬 worktree 的 subagents 修復問題。在開始每一個新批次之前，先將相關修復合併回當前分支。
8. 確認所有修復已經合併，且所有修復用 worktree 已經被清理。

如果使用了 subagents 加速修復流程，subagents 需要嚴格按照以下流程工作：
1. 識別修復範圍，閱讀相關代碼檔案。
2. 根據問題複雜度選擇修復方式：簡單修復直接編輯，複雜修復使用 `systematic-debug`。
3. 使用 `commit` 技能，將修復提交到所屬 worktree。

### 4. 回歸測試

為修復添加回歸測試，避免同樣的問題在未來再次出現。

### 5. 驗證修復

所有修復完成後，執行驗證：
- 執行專案測試套件，確認無回歸（regression）
- 確認修復後的代碼可正常編譯/通過 lint
- 若有相關手動測試情境，一併確認

驗證未通過的修復必須重新進入修復流程。

### 6. 產出修復摘要

記錄本次修復的摘要，包含：
- 已修復的問題列表（含 P0-P3 分級）
- 每個問題的修復方式與涉及檔案
- 驗證結果（測試通過 / 編譯成功）
